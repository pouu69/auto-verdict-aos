# Google Play Console 등록 — 처음부터 따라하기 (친절판)

> AutoVerdict v1.0.0를 Play Store에 올리기 위한 **클릭 단위 가이드**.
> 메뉴를 못 찾을 때 이 문서의 "📍 메뉴 위치"를 보고 그대로 따라가면 됩니다.
> 입력값 모음·체크리스트는 [play-store-checklist.md](./play-store-checklist.md) 참고.

---

## 🗺 Play Console 화면 구조 먼저 이해하기

Play Console은 **2단계**로 나뉩니다. 헷갈리는 이유가 이 구조 때문입니다.

```
1) 전체 계정 화면 (모든 앱 목록)
     └─ "앱 만들기" 버튼

2) 개별 앱 화면 (앱을 클릭하면 들어감)
     └─ 왼쪽 사이드바에 모든 메뉴가 있음
         ├─ 대시보드        ← 여기 "할 일 체크리스트"가 핵심
         ├─ 테스트 및 출시   ← AAB 업로드, 테스트 트랙
         ├─ 모니터링 및 개선
         ├─ 수익 창출
         ├─ 성장
         └─ 정책 및 프로그램  ← "앱 콘텐츠"(정책 설문)가 여기
```

> 💡 **핵심 팁**: 앱을 만들고 나면 **대시보드**에 "앱 설정" + "출시" 두 개의 할 일 카드가 뜹니다.
> 메뉴를 헤맬 필요 없이 **그 카드의 항목을 위에서 아래로 하나씩 클릭**하면 됩니다.
> 아래 STEP들도 그 순서를 따릅니다.

## STEP 0 — Keystore 백업 (5분, 제일 먼저!)

⚠️ 이 파일을 잃으면 **앱 업데이트가 영원히 불가능**합니다. 등록보다 먼저 하세요.


## STEP 1 — 개인정보처리방침 공개 URL 만들기 (30분)

Play Console은 앱 내 화면이 아니라 **인터넷에 공개된 URL**을 요구합니다.

### 가장 쉬운 방법 — GitHub Gist (5분)
1. https://gist.github.com 접속 → 로그인
2. 파일 이름 `privacy-policy.md` 입력
3. `docs/privacy-policy.md` 내용 전체 붙여넣기
4. **"Create public gist"** 클릭
5. 우측 상단 **Raw** 버튼 클릭 → 그 주소를 복사 (이게 공개 URL)

### 또는 GitHub Pages / Notion
- Notion: 페이지에 내용 붙여넣기 → 우측 상단 **공유 → 웹에 게시 → 링크 복사**

➡️ 만든 URL을 메모해 두세요. STEP 4의 "앱 콘텐츠"에서 입력합니다.

## STEP 2 — 스크린샷 캡처 (1~2시간, 최소 2장)

휴대폰 스크린샷이 있어야 스토어 등록이 완료됩니다.

```bash
# 1) 에뮬레이터나 실기기 연결 확인
adb devices

# 2) 디버그 앱 설치
export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
./gradlew installDebug

# 3) 화면을 띄운 뒤 캡처 (원하는 화면마다 반복)
adb exec-out screencap -p > docs/assets/screenshots/01.png
```
추천 화면: 온보딩 → 분석 입력 → 분석 결과 → 저장 목록 → 비교 화면.
규격: PNG/JPG, 가로·세로 320~3840px.

## STEP 3 — Play Console 계정 + 앱 만들기

### 3-1. 개발자 계정 등록 (최초 1회)
- https://play.google.com/console 접속
- **25 USD 1회 결제** + 신원 확인(여권/신분증)
- 검증에 **1~3일** 걸릴 수 있음 (이 시간은 기다려야 함)

### 3-2. 앱 만들기
📍 **메뉴 위치**: 전체 화면 우측 상단 **"앱 만들기"** 버튼

| 입력 | 값 |
|---|---|
| 앱 이름 | AutoVerdict — 엔카 매물 평가 |
| 기본 언어 | 한국어 (대한민국) |
| 앱 / 게임 | 앱 |
| 무료 / 유료 | 무료 |
| 선언 2개 | 모두 체크 |

→ 만들면 자동으로 **그 앱의 대시보드**로 들어갑니다. 여기서부터가 본 게임.

---

## STEP 4 — 대시보드 "앱 설정" 카드 (정책 설문)

📍 **메뉴 위치**: 왼쪽 사이드바 맨 위 **대시보드** → 가운데 **"앱 설정"** 카드
(또는 사이드바 **정책 및 프로그램 → 앱 콘텐츠**)

아래 항목이 목록으로 보입니다. **위에서부터 하나씩** 클릭해 채우세요.

| 설문 항목 | 응답 |
|---|---|
| 앱 액세스 권한 | "모든 기능을 제한 없이 사용 가능" |
| 광고 | **예, 광고 포함** (AdMob 사용) |
| 콘텐츠 등급 | 설문 시작 → 이메일 입력 → 카테고리 "유틸리티" → 질문 **모두 아니오** → 제출 (전체이용가) |
| 타겟층 및 콘텐츠 | 만 13세 이상 선택 / 아동 대상 아님 |
| 데이터 보안 | 아래 4-1 참고 |
| 정부 앱 | 아니오 |
| 금융 기능 | 아니오 |
| 건강 | 해당 없음 |

### 4-1. 데이터 보안(Data safety) 입력값
- **데이터 수집/공유**: 예 → **기기 또는 기타 ID → 광고 ID** 1개만 체크
- 수집 목적: **광고 또는 마케팅**
- 전송 중 암호화: **예 (HTTPS)**
- 삭제 요청 가능: **예**
> 자세한 응답은 `docs/store-listing.md` 9절에 정리돼 있음.

## STEP 6 — 앱 서명 확인 (그냥 켜두면 됨)

📍 **메뉴 위치**: 사이드바 **테스트 및 출시 → 설정 → 앱 서명**
- "Play 앱 서명"은 **기본으로 켜져 있음** → 그대로 두기.
- 우리 `autoverdict-upload.jks`는 "업로드 키" 역할. 첫 AAB 올리면 자동 연결됩니다.
> ⚠️ 여기 있는 "앱 무결성/Play Integrity"는 **건드릴 필요 없음** (이 앱엔 불필요).

## STEP 7 — 내부 테스트로 첫 업로드

📍 **메뉴 위치**: 사이드바 **테스트 및 출시 → 테스트 → 내부 테스트**

1. 우측 **"새 버전 만들기"** 클릭
2. **App bundle** 영역에 `app-release.aab` 드래그 업로드
3. 출시명/출시 노트 입력 (노트는 store-listing의 "무엇이 새로운가요?" 사용)
4. **저장 → 검토 → 내부 테스트 트랙으로 출시**
5. "테스터" 탭에서 **본인 이메일** 등록 → 받은 링크로 직접 설치 테스트

> 내부 테스트는 보통 수십 분~수 시간 내 반영. 여기서 앱이 잘 깔리고 도는지 먼저 확인.

## STEP 8 — 비공개 테스트 (신규 개발자 의무)

⚠️ 2023년 이후 **신규 개인 개발자**는 프로덕션 전에:
**테스터 12명 이상 × 14일 이상** 비공개 테스트가 **의무**입니다.

📍 **메뉴 위치**: **테스트 및 출시 → 테스트 → 비공개 테스트** → 새 트랙 생성

- 내부 테스트의 AAB를 이 트랙으로 **승격**하거나 새로 업로드
- 이메일 리스트 또는 Google 그룹으로 **12명 이상** 초대
- **14일 동안 실제로 앱을 사용**해야 카운트됨 (설치만 하면 인정 안 됨)
- 모집 팁: 가족·지인·동료 → Reddit r/AndroidApps, 개발자 디스코드

---

## STEP 9 — 프로덕션 출시 (실제 게시)

📍 **메뉴 위치**: **테스트 및 출시 → 프로덕션**

조건 충족(12명·14일) 후:
1. **"새 버전 만들기"** → 비공개 트랙의 AAB를 승격
2. 출시 국가: **대한민국** 선택
3. **검토를 위해 제출** → Google 심사 (보통 1~7일) → 통과 시 자동 게시 🎉
